
<h1 align="center">SOC Log Analysis using SIEM</h1>

<p align="center">
A practical Security Operations Center (SOC) project demonstrating how security logs can be collected,
analyzed, and monitored using a Security Information and Event Management (SIEM) platform.
</p>

<hr>

<h2>Project Overview</h2>

<p>
This project focuses on analyzing security logs generated by systems and networks to detect suspicious
activities using SIEM tools.
</p>

<p>
Security logs were monitored and analyzed using <b>Splunk SIEM</b> to identify potential threats such as
unauthorized login attempts and abnormal network behavior.
</p>

<p>
The project demonstrates how SOC analysts investigate logs and create detection rules for identifying
security incidents.
</p>

<hr>

<h2>Lab Architecture</h2>

<pre>
Windows System
     |
     |  Security Event Logs
     |

<h2>Log Collection</h2>

<p>
System logs were collected from Windows security logs and ingested into the Splunk platform
for centralized monitoring.
</p>

<p>
These logs include authentication events, system activity, and network access attempts.
</p>

<hr>

<h2>Security Log Investigation</h2>

<p>
One of the key logs analyzed in this project was the Windows security event:
</p>

<p><b>Event ID: 4625</b></p>

<p>
Meaning: <b>Failed login attempt</b>
</p>

<p>
Repeated occurrences of this event may indicate a potential brute force attack.
</p>

<hr>

<h2>Splunk Log Analysis</h2>

<p>
Splunk dashboards were used to visualize security events and detect suspicious activity.
</p>

<p>
Example Splunk query used to detect failed logins:
</p>

<pre>
EventCode=4625
</pre>

<p>
This query identifies all failed authentication attempts recorded in system logs.
</p>

<hr>

<h2>Alert Rule Configuration</h2>

<p>
Custom alert rules were created to detect suspicious behavior such as excessive login failures
or abnormal network scanning.
</p>

<p>
Example detection logic:
</p>

<pre>
index=security_logs EventCode=4625
| stats count by src_ip
| where count > 5
</pre>

<p>
If multiple failed login attempts occur within a short period, an alert is triggered
to notify security analysts.
</p>
